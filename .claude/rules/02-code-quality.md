# Code Quality and Structure

## Clean Code
- Write clear, concise, readable, and well-documented code
- Follow established clean code principles

## Modularity & Separation of Concerns
- Structure code into logical modules with clear responsibilities
- Define well-defined interfaces
- Minimize coupling and maximize cohesion

## Consistency
- Maintain consistency in coding style, naming conventions, and architectural patterns
- Use standard linters and formatters for the chosen language

## Descriptive Naming
- Use clear and descriptive names for variables, functions, classes, modules
- Names should accurately reflect their purpose

## Language Best Practices
- Adhere strictly to idiomatic conventions and best practices of the specific programming language
- Utilize modern language features effectively

## File Organization
- Avoid circular dependencies
- Keep files under 500 lines if possible
- Avoid using the same file for different features, components, services, models, or interfaces
- Avoid duplicate functionality across different files (e.g., same function in feature and page in React)

## Documentation
- Add task IDs from tasks.md to implemented files for tracking
- Write meaningful comments to explain the "why" behind complex logic
