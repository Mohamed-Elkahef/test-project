# Architecture and Design

## Appropriate Architecture
- Select and implement architectural patterns that fit project requirements, scale, and team capabilities

## API-First Design
- Design APIs thoughtfully, considering consumers, versioning, and documentation
- Use OpenAPI specification where applicable

## Data Management
- Choose appropriate data storage solutions (SQL, NoSQL, caches) based on data characteristics and access patterns
- Ensure data integrity, security, and efficient querying

## Frontend Architecture (React)
- Use Vite for React applications
- Use JSX (not JS) for React components
- Frontend should call service layer and get/post data from mock folder containing JSON test data

## Task Tracking
- Always add the task ID from tasks.md to implemented files for easy tracking
